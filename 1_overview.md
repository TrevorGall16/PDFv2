(Ce document définit la vision et le "quoi".)

1. Project Type Système de génération de documents utilitaires haute performance (HTML/CSS vers PDF) conçu pour l'impression physique A4.

2. One-Sentence Summary Un protocole de survie visuel multilingue pour réfrigérateur, conçu pour neutraliser les pulsions alimentaires nocturnes des sportifs par des alternatives de satiété immédiates.

3. Core Purpose L'objectif est de supprimer la fatigue décisionnelle entre 20h et 22h en offrant un "chemin de moindre résistance" vers des choix sains, transformant une pulsion émotionnelle en action rationnelle.

4. Target Users Pratiquants de musculation et de fitness ayant une discipline stricte en journée mais sujets aux craquages nocturnes (ennui, stress, habitude).

5. Feature List (Input → Output)

Système Multilingue Dynamique → Input : Fichier JSON → Output : Rendu instantané en Français, Anglais ou toute langue ajoutée.

Matrice de "Swaps" (Cœur du produit) → Input : Liste d'envies (sucré/gras) → Output : Alternatives de satiété (ex: Pomme, Fromage blanc) avec bénéfices associés.

Checklist de Sécurité Psychologique → Input : Étapes de vérification (eau, fatigue) → Output : Validation avant action.

Zone d'Engagement (Modulaire) → Output : Espace dédié aux objectifs du lendemain (peut s'étendre sur une Page 2 si nécessaire).

6. Constraints & Priorities

Modularité Totale : Le projet doit être construit par blocs indépendants (CSS Grid/Flexbox). Déplacer ou supprimer une section ne doit pas briser le reste de la mise en page.

Flexibilité de Pagination : Bien que l'objectif soit 1 page (A4), l'IA est autorisée à déborder sur une 2ème page si la lisibilité ou le volume de texte le nécessite.

Qualité d'Impression : Rendu vectoriel via HTML/CSS pour une netteté parfaite des textes à 300 DPI.

Standardisation des Couleurs : Ne jamais utiliser de noir pur (#000000). Utiliser des gris profonds pour un aspect premium.

7. Success Criteria

Le document est parfaitement lisible à 2 mètres de distance.

Le changement de langue dans le fichier de données met à jour tout le document instantanément sans erreur de layout.

Le design ne contient aucun "fluff" inutile : chaque élément doit servir la discipline de l'utilisateur.
